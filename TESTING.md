## Запуск локально

Просто откройте файл в браузере:

```
miniapps/tma.html
```

Telegram WebApp API не работает в браузере — используется mock-объект.  
Онбординг при первом запуске сохраняется через `localStorage`.

## Деплой

Проект деплоится автоматически при пуше в `master` ветку GitHub.  
Vercel подхватывает изменения за ~30 секунд.

Роутинг (`vercel.json`):
- `/` → `index.html`
- `/tma` → `miniapps/tma.html`

## Известные ограничения

| Что | Почему |
|---|---|
| MainButton / BackButton не видны в браузере | Это нативные кнопки Telegram, работают только внутри Telegram |
| Онбординг в браузере запоминается через `localStorage` | В Telegram используется `CloudStorage` |
| Telegram v6.0 не поддерживает `BackButton`, `CloudStorage`, `enableClosingConfirmation` | Все вызовы обёрнуты в `try/catch`, фолбэк на `localStorage` |

## Частые ошибки и решения

| Ошибка | Решение |
|---|---|
| Кнопки не работают | Открыть консоль (`F12`) → проверить ошибки JS |
| Фото не загружается | Убедиться что `photo.jpg` лежит в корне проекта |
| Заявка не приходит | Проверить токен бота в переменной `TG_TOKEN` в `tma.html` (строка ~1068) |
| Онбординг показывается каждый раз | Очистить `localStorage`: в консоли `localStorage.removeItem('onboard_done')` |
| `/tma` открывает лендинг | Проверить `vercel.json`, убедиться что роут `/tma` ведёт на `/miniapps/tma.html` |
